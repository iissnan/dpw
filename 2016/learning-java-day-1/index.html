Data Types

Java has 8 basic data types that store integers, floating-point numbers, characters, and boolean values. These often are called primitive types because they are built-in parts of the language rather than objects, which makes them easier to create and use. These data types have the same size and characterstics no matter what operating system and platform you’re on, unlike some data types in other programming languages.

8 primitive types

Type Description
byte 8 bits, -128 to 127
short 16 bits, -32,768 to 32,767
int 32 bits, -2,147,483,648 to 2,147,483,647
long 64 bits, -9,223,374,036,854,775,808 to 9,223,374,036,854,775,807
float 1.4E-45 to 3.4E+38. Numbers with a decimal point(Floating-point Number)
double 4.9-324 to 1.7E+308. More precision than float (preferred)
char For individual characters, such as letters, numbers, punctuation etc.
boolean true or false

All these variable types appear in lowercase. Some classes have the same names as these data types, but with different capitalization, such as Boolean and Double. These are created and referenced differently in Java.

void

There’s actually a 9th primitive date type in Java, void, which represents nothing. It’s used in methods to indicate that they do not return a value.

Literals

A literal is any number, text, or other information that directly represents a value.

Boolean Literals

The Boolean literals true and false are the only two values you can use when assigning a value to a boolean variable type or using a Boolean in a statement.

Caution
You must use the value true and false to represent Boolean values. A value of 1 is not equivalent to true and 0 is not equivalent to false.

String Literals

A string in Java is an object rather than a primitive date type. Strings are not stored in arrays as they are in languages such as C. Because string objects are real objects in Java, methods are available to combine strings, modify strings and determine whether two strings have the same value.

Although string literals are used in a manner similar to other literals in a program, they are handled differently behind the scenes. With a string literal, Java stores that value as a String object. You don’t have to explicitly create a new object, so they are as easy to work with as primitive data types.

Logical Operators

Expressions that result in Boolean values, such as comparision operations, can be combined to form more complex expressions. This is handled through logical operators, which are used for the logical combinations AND, OR, XOR, and logical NOT.

For AND combinations, the & or && logical operator is used. The difference between & and && lies in how much work Java does on the combined expression. If & is used, the expressions on both siders of the & are evaluated no matter what. If && is used and the left side is false, the expression on the right side of the && never is evaluated.

For OR combinations, the | or || logical operator is used.

The XOR combination has one logical operator, ^. This results in a true value only if the Boolean expressions it cobines have opposite values. If both are true or both are false, the & operator produces a false value.

The NOT combination use the ! logical operator followed by a single expression. It reverses the value of a Boolean expression in the same way that a minus sign reverses the positive or negative sign on a number.
